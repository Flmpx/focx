🔶 OASet_M - 开放地址哈希集合（多种数据类型）

采用开放定址法解决哈希冲突的集合存储，键可存储任意类型数据

📖 概述

OASet_M (Open Addressing Set Multiple Data) 采用开放定址法（Open Addressing）解决哈希冲突。当发生冲突时，使用线性探测（Linear Probing）寻找下一个可用的槽位。

⚙️ 数据类型定义

// 集合条目
typedef struct Entry_M_inOASet {
    Data_M key;      // 键
    int state;       // 状态
    bool isEmpty;    // 是否为空
} Entry_M_inOASet;

// 哈希集合
typedef struct OASet_M {
    Entry_M_inOASet* arr;    // 哈希表数组
    int mod;                 // 模数
    int len;                 // 当前元素数量
    int size;                // 哈希表容量
} OASet_M;

🚀 快速开始

#include "base.h"
#include "Set/OASet/Multiple_Data/oaset_mdata.h"
#include "Oper/Int_Info/int_info.h"
#include "Oper/String_Info/string_info.h"

int main() {
    // 1. 初始化集合
    OASet_M set;
    initMOASet(&set);

    // 2. 插入键
    int num = 1;
    Data_M key1 = Data_M_OWN(&num, NULL, &Info_Int, 0);
    insertMKeyInMOASet(&set, key1);

    char str[] = "hello";
    Data_M key2 = Data_M_OWN(str, NULL, &Info_String, 0);
    insertMKeyInMOASet(&set, key2);

    // 3. 打印集合
    printMOASet(&set);

    // 4. 释放集合
    freeMOASet(&set);
    return 0;
}

⚠️ 注意事项

  • Set 只存储唯一键，不允许重复插入
  • 通过 getCopy 系列函数获取的数据使用后必须释放
  • 每次使用完集合后必须调用 freeMOASet 释放整个集合
  • OASet 使用开放定址法，适合数据量较小且分布均匀的场景
