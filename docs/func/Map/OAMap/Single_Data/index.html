💠 OAMap_S - 开放地址哈希表（单一数据类型）

采用开放定址法解决哈希冲突的键值对存储，需要提供键和值的数据操作集

📖 概述

OAMap_S (Open Addressing Map Single Data) 采用开放定址法（Open Addressing）解决哈希冲突。当发生冲突时，使用线性探测（Linear Probing）寻找下一个可用的槽位。

⚙️ 数据类型定义

// 键值对条目
typedef struct Entry_S_inOAMap {
    Data_S key;      // 键
    Data_S val;      // 值
    int state;       // 状态
    bool isEmpty;    // 是否为空
} Entry_S_inOAMap;

// 哈希表
typedef struct OAMap_S {
    Entry_S_inOAMap* arr;    // 哈希表数组
    int mod;                 // 模数
    int len;                 // 当前元素数量
    int size;                // 哈希表容量
    InfoOfData* keyInfo;     // 键的操作集
    InfoOfData* valInfo;     // 值的操作集
} OAMap_S;

🚀 快速开始

#include "base.h"
#include "Map/OAMap/Single_Data/oamap_sdata.h"
#include "Oper/Int_Info/int_info.h"

int main() {
    // 1. 初始化哈希表
    OAMap_S map;
    initSOAMap(&map, &Info_Int, &Info_Int);

    // 2. 插入键值对
    insertSKeyAndSValInSOAMap(&map, Data_S_OWN(1, NULL), Data_S_OWN(100, NULL));
    insertSKeyAndSValInSOAMap(&map, Data_S_OWN(2, NULL), Data_S_OWN(200, NULL));

    // 3. 查找值
    Data_S val = getCopySValBySKeyInSOAMap(&map, Data_S_OWN(1, NULL));
    if (!val.isEmpty) {
        printf("Key 1's value: %d\n", *(int*)val.data);
        freeSValInSOAMap(&map, &val);
    }

    // 4. 释放哈希表
    freeSOAMap(&map);
    return 0;
}

⚠️ 注意事项

  • 通过 getCopy 系列函数获取的数据使用后必须释放
  • 通过 getPtr 系列函数获取的数据指针直接指向哈希表内部，不要释放
  • 每次使用完哈希表后必须调用 freeSOAMap 释放整个哈希表
  • OAMap 使用开放定址法，适合数据量较小且分布均匀的场景
