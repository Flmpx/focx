➡️ SList_S - 单向链表（单一数据类型）

存储单一类型数据的单向链表，需要提供数据操作集 (InfoOfData)

📖 概述

SList_S (Single List Single Data) 是一种单向链表数据结构，只能存储单一类型的数据。所有数据都需要提供对应的 InfoOfData 操作集指针，用于执行复制、比较、释放等操作。

说明: SList 与 DList 的区别在于 SList 是单向的，只能从头向尾遍历，不能反向遍历

⚙️ 数据类型定义

// 链表节点
typedef struct Node_S_inSList {
    struct Node_S_inSList* next;   // 后继节点
    Data_S val;                    // 数据
} Node_S_inSList;

// SList_S 链表
typedef struct {
    Node_S_inSList* head;      // 头节点
    Node_S_inSList* tail;      // 尾节点
    InfoOfData* valInfo;       // 数据操作集
    int size;                  // 链表长度
} SList_S;

🚀 快速开始

#include "base.h"
#include "List/SList/Single_Data/slist_sdata.h"
#include "Oper/Int_Info/int_info.h"

int main() {
    // 1. 初始化链表
    SList_S list;
    initSSList(&list, &Info_Int);

    // 2. 创建数据并插入
    int i;
    for (i = 0; i < 10; i++) {
        insertSValAtEndInSSList(&list, Data_S_OWN(&i, NULL), Data_Copy);
    }

    // 3. 释放链表
    freeSSList(&list);
    return 0;
}

🔍 查询操作

Data_S getSValByPosINSSList(SList_S* plist, int pos, selectOfCopy isCopyVal)

查找在单类型单向链表的指定位置的值

参数:
  • plist - 链表指针
  • pos - 位置（从 0 开始）
  • isCopyVal - 返回的值是否要复制
返回: 返回找到的值，若没找到，返回空值（用字段 isEmpty 检查）
说明: 如果不复制，可用于修改内部数据；如果复制，会创建副本，副本默认拥有数据所有权，使用完后通过 freeSValInSSList 函数释放

判断单类型单向链表中是否有当前值

参数:
  • plist - 链表指针
  • val - 待查找的值
返回: 如果存在返回 true，否则 false

⚠️ 注意事项

  • SList 是单向链表，只能从头向尾遍历，不能反向遍历
  • 使用 Data_Copy 模式插入数据时，链表会自动复制一份数据
  • 使用 Data_NoCopy 模式插入数据时，链表只保存指针
  • 每次使用完链表后必须调用 freeSSList 释放整个链表
