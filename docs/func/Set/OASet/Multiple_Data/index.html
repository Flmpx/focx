📍 OASet_M - 开放寻址集合（多数据类型）

采用开放寻址法处理冲突的哈希集合，存储任意类型的元素

📖 概述

OASet_M (Open Addressing Set Multiple Data) 是一种开放寻址哈希集合数据结构，采用开放寻址法处理哈希冲突。它可以存储任意类型的元素，每条元素都自带 InfoOfData 操作集指针。

⚙️ 数据类型定义

// 条目类型
typedef struct Entry_M_inOASet {
    Data_M key;      // 键
    int state;       // 状态（用于标记删除或占用）
    bool isEmpty;    // 是否为空
} Entry_M_inOASet;

// OASet_M 集合
typedef struct OASet_M {
    Entry_M_inOASet* arr;    // 哈希桶数组
    int mod;                 // 哈希模数
    int len;                 // 哈希表长度
    int size;                // 元素数量
} OASet_M;

🚀 快速开始

#include "base.h"
#include "Set/OASet/Multiple_Data/oaset_mdata.h"
#include "Oper/Int_Info/int_info.h"
#include "Oper/String_Info/string_info.h"

int main() {
    // 1. 初始化集合
    OASet_M set;
    initMOASet(&set);

    // 2. 插入不同类型的元素
    int id = 1;
    char* name = "Alice";
    
    insertMKeyInMOASet(&set, Data_M_OWN(&id, NULL, &Info_Int, TYPE_INT), Data_Copy);
    insertMKeyInMOASet(&set, Data_M_OWN(name, NULL, &Info_String, TYPE_STRING), Data_Copy);

    // 3. 释放集合
    freeMOASet(&set);
    return 0;
}

📋 函数列表

🔧 初始化与销毁

void initMOASet(OASet_M* pSet)

初始化多类型开放寻址集合

参数:
  • pSet - 集合指针
void freeMKeyInMOASet(Data_M* key)

释放多类型开放寻址集合中的键

参数:
  • key - 待释放的键
void freeMOASet(OASet_M* pSet)

释放多类型开放寻址集合

参数:
  • pSet - 集合指针

➕ 插入操作

InfoOfReturn insertMKeyInMOASet(OASet_M* pSet, Data_M key, selectOfCopy isCopyKey)

在多类型开放寻址集合中插入键

参数:
  • pSet - 集合指针
  • key - 待插入的键
  • isCopyKey - 是否要复制传入的 key
返回: 操作结果状态码

🔍 查询操作

Data_M getMKeyByMKeyInMOASet(OASet_M* pSet, Data_M key, selectOfCopy isCopyKey)

通过键找到多类型开放寻址集合中的键

参数:
  • pSet - 集合指针
  • key - 待查找的键
  • isCopyKey - 返回的键是否要复制
返回: 返回找到的键，若没找到，返回空键（用字段 isEmpty 检查）
说明: 如果不复制，可用于修改内部数据；如果复制，会创建副本，副本默认拥有数据所有权，使用完后通过 freeMKeyInMOASet 函数释放
bool hasMKeyInMOASet(OASet_M* pSet, Data_M key)

判断多类型开放寻址集合中是否存在当前键

参数:
  • pSet - 集合指针
  • key - 待查找的键
返回: 如果存在返回 true，否则 false

➖ 删除操作

InfoOfReturn delMKeyByMKeyInMOASet(OASet_M* pSet, Data_M key)

删除多类型开放寻址集合中的键

参数:
  • pSet - 集合指针
  • key - 待删除的键
返回: 操作结果状态码

⚠️ 注意事项

  • Set 中的元素唯一，不允许重复
  • OASet_M 每条元素都自带 InfoOfData，可以存储多种不同类型的元素
  • OASet 采用开放寻址法处理哈希冲突，空间利用率高
  • 每次使用完集合后必须调用 freeMOASet 释放整个集合
