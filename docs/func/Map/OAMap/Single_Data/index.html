📍 OAMap_S - 开放寻址哈希表（单一数据类型）

采用开放寻址法处理冲突的哈希表，存储单一类型的键值对

📖 概述

OAMap_S (Open Addressing Map Single Data) 是一种开放寻址哈希表数据结构，采用开放寻址法处理哈希冲突。它只能存储单一类型的键和值，需要提供键和值的数据操作集 InfoOfData。

⚙️ 数据类型定义

// 条目类型
typedef struct Entry_S_inOAMap {
    Data_S key;      // 键
    Data_S val;      // 值
    int state;       // 状态（用于标记删除或占用）
    bool isEmpty;    // 是否为空
} Entry_S_inOAMap;

// OAMap_S 哈希表
typedef struct OAMap_S {
    Entry_S_inOAMap* arr;    // 哈希桶数组
    int mod;                 // 哈希模数
    int len;                 // 哈希表长度
    int size;                // 元素数量
    InfoOfData* keyInfo;     // 键的数据操作集
    InfoOfData* valInfo;     // 值的数据操作集
} OAMap_S;

🚀 快速开始

#include "base.h"
#include "Map/OAMap/Single_Data/oamap_sdata.h"
#include "Oper/Int_Info/int_info.h"
#include "Oper/String_Info/string_info.h"

int main() {
    // 1. 初始化哈希表
    OAMap_S map;
    initSOAMap(&map, &Info_Int, &Info_String);

    // 2. 插入键值对
    char* name1 = "Alice";
    char* name2 = "Bob";
    insertSKeyAndSValInSOAMap(&map, 
        Data_S_OWN(1, NULL), Data_Copy, 
        Data_S_OWN(name1, NULL), Data_Copy);
    insertSKeyAndSValInSOAMap(&map, 
        Data_S_OWN(2, NULL), Data_Copy, 
        Data_S_OWN(name2, NULL), Data_Copy);

    // 3. 释放哈希表
    freeSOAMap(&map);
    return 0;
}

🔍 查询操作

参数:
  • pMap - 哈希表指针
  • key - 待查找的键

⚠️ 注意事项

  • OAMap 采用开放寻址法处理哈希冲突，空间利用率高
  • 使用 Data_Copy 模式插入数据时，哈希表会自动复制一份数据
  • 每次使用完哈希表后必须调用 freeSOAMap 释放整个哈希表
