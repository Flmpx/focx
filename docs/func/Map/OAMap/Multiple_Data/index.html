💎 OAMap_M - 开放地址哈希表（多种数据类型）

采用开放定址法解决哈希冲突的键值对存储，键和值都可存储任意类型数据

📖 概述

OAMap_M (Open Addressing Map Multiple Data) 采用开放定址法（Open Addressing）解决哈希冲突。当发生冲突时，使用线性探测（Linear Probing）寻找下一个可用的槽位。

⚙️ 数据类型定义

// 键值对条目
typedef struct Entry_M_inOAMap {
    Data_M key;      // 键
    Data_M val;      // 值
    int state;       // 状态
    bool isEmpty;    // 是否为空
} Entry_M_inOAMap;

// 哈希表
typedef struct OAMap_M {
    Entry_M_inOAMap* arr;    // 哈希表数组
    int mod;                 // 模数
    int len;                 // 当前元素数量
    int size;                // 哈希表容量
} OAMap_M;

🚀 快速开始

#include "base.h"
#include "Map/OAMap/Multiple_Data/oamap_mdata.h"
#include "Oper/Int_Info/int_info.h"
#include "Oper/String_Info/string_info.h"

int main() {
    // 1. 初始化哈希表
    OAMap_M map;
    initMOAMap(&map);

    // 2. 插入键值对
    int key1 = 1;
    char* val1 = "hello";
    Data_M data1 = Data_M_OWN(&key1, NULL, &Info_Int, 0);
    Data_M data2 = Data_M_OWN(val1, NULL, &Info_String, 0);
    insertMKeyAndMValInMOAMap(&map, data1, data2);

    // 3. 释放哈希表
    freeMOAMap(&map);
    return 0;
}

⚠️ 注意事项

  • 通过 getCopy 系列函数获取的数据使用后必须释放
  • 通过 getPtr 系列函数获取的数据指针直接指向哈希表内部，不要释放
  • 每次使用完哈希表后必须调用 freeMOAMap 释放整个哈希表
  • OAMap 使用开放定址法，适合数据量较小且分布均匀的场景
