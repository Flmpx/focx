🗝️ ChainMap_M - 链式哈希表（多种数据类型）

采用链表法解决哈希冲突的键值对存储，键和值都可存储任意类型数据

📖 概述

ChainMap_M (Chain Map Multiple Data) 采用链表法（Separate Chaining）解决哈希冲突。每个哈希桶都是一个链表，可以在同义词冲突时将多个键值对链接在一起。

⚙️ 数据类型定义

// 键值对条目
typedef struct Entry_M_inChainMap {
    Data_M key;      // 键
    Data_M val;      // 值
    bool isEmpty;    // 是否为空
} Entry_M_inChainMap;

// 哈希表
typedef struct ChainMap_M {
    List_M_inChainMap* arr;    // 哈希桶数组
    int mod;                   // 模数（哈希表大小）
    int len;                   // 当前元素数量
    int size;                  // 哈希表容量
} ChainMap_M;

🚀 快速开始

#include "base.h"
#include "Map/ChainMap/Multiple_Data/chainmap_mdata.h"
#include "Oper/Int_Info/int_info.h"
#include "Oper/String_Info/string_info.h"

int main() {
    // 1. 初始化哈希表
    ChainMap_M map;
    initMChainMap(&map);

    // 2. 插入键值对
    int key1 = 1;
    char* val1 = "hello";
    Data_M data1 = Data_M_OWN(&key1, NULL, &Info_Int, 0);
    Data_M data2 = Data_M_OWN(val1, NULL, &Info_String, 0);
    insertMKeyAndMValInMChainMap(&map, data1, data2);

    // 3. 释放哈希表
    freeMChainMap(&map);
    return 0;
}

⚠️ 注意事项

  • 通过 getCopy 系列函数获取的数据使用后必须释放
  • 通过 getPtr 系列函数获取的数据指针直接指向哈希表内部，不要释放
  • 每次使用完哈希表后必须调用 freeMChainMap 释放整个哈希表
  • ChainMap 使用链表法解决冲突，适合数据量较大或分布不均匀的场景
