🔑 ChainMap_S - 链式哈希表（单一数据类型）

采用链表法解决哈希冲突的键值对存储，需要提供键和值的数据操作集

📖 概述

ChainMap_S (Chain Map Single Data) 采用链表法（Separate Chaining）解决哈希冲突。每个哈希桶都是一个链表，可以在同义词冲突时将多个键值对链接在一起。

⚙️ 数据类型定义

// 键值对条目
typedef struct Entry_S_inChainMap {
    Data_S key;      // 键
    Data_S val;      // 值
    bool isEmpty;    // 是否为空
} Entry_S_inChainMap;

// 哈希表
typedef struct ChainMap_S {
    List_S_inChainMap* arr;    // 哈希桶数组
    InfoOfData* keyInfo;       // 键的操作集
    InfoOfData* valInfo;       // 值的操作集
    int mod;                   // 模数（哈希表大小）
    int len;                   // 当前元素数量
    int size;                  // 哈希表容量
} ChainMap_S;

🚀 快速开始

#include "base.h"
#include "Map/ChainMap/Single_Data/chainmap_sdata.h"
#include "Oper/Int_Info/int_info.h"  // 键和值都用整数
#include "Oper/String_Info/string_info.h"  // 或者用字符串

int main() {
    // 1. 初始化哈希表
    ChainMap_S map;
    initSChainMap(&map, &Info_Int, &Info_Int);

    // 2. 插入键值对
    insertSKeyAndSValInSChainMap(&map, Data_S_OWN(1, NULL), Data_S_OWN(100, NULL));
    insertSKeyAndSValInSChainMap(&map, Data_S_OWN(2, NULL), Data_S_OWN(200, NULL));

    // 3. 查找值
    Data_S val = getCopySValBySKeyInSChainMap(&map, Data_S_OWN(1, NULL));
    if (!val.isEmpty) {
        printf("Key 1's value: %d\n", *(int*)val.data);
        freeSValInSChainMap(&map, &val);
    }

    // 4. 释放哈希表
    freeSChainMap(&map);
    return 0;
}

⚠️ 注意事项

  • 通过 getCopy 系列函数获取的数据使用后必须释放
  • 通过 getPtr 系列函数获取的数据指针直接指向哈希表内部，不要释放
  • 每次使用完哈希表后必须调用 freeSChainMap 释放整个哈希表
  • ChainMap 使用链表法解决冲突，适合数据量较大或分布不均匀的场景
